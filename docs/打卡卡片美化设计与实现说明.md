# 打卡卡片美化设计与实现说明

## 1. 文档目的

本文用于说明当前“小程序打卡卡片”如何从基础记录卡，升级为更有“邮件感”“寄送感”“收藏感”的视觉卡片，并沉淀本轮已经实现的美化方法、设计原则、技术实现和后续优化方向。

本文关注的问题包括：

- 当前打卡卡片为什么会显得偏简单
- 参考图中的视觉质感主要来自哪些元素
- 这次具体增加了哪些邮件元素
- 小程序 canvas 场景下如何落地这些美化效果
- 后续如何继续把卡片做得更稳定、更适合生产环境

## 2. 当前卡片的原始状态

原始版本的打卡卡片已经具备基础能力：

- 纸张底色
- 标题与任务文字
- 用户记录与陪伴文案
- 图片拼接
- 日期、地点、条码、邮戳等基础元素

但视觉上仍有几个明显问题：

- 邮件元素偏少，更像普通海报卡片，而不是“寄出的记录卡”
- 顶部信息结构比较平，没有“邮票区 / 地址区 / 邮戳区”的信件层次
- 图片虽然有胶带和纸面，但整体收藏感还不够强
- 底部信息偏功能性，缺少“投递完成”的仪式感

也就是说，原始卡片已经“能用”，但离“好看、有记忆点、适合分享”还有距离。

## 3. 参考图带来的设计方向

参考图 `docs/打卡卡片参考.jpg` 的质感，不是来自复杂插画，而是来自一组非常典型的“邮件 / 航空信 / 明信片”元素组合：

- 左上角邮票
- 邮票齿孔边缘
- 多个不同颜色和角度的邮戳 / 签收章 / 分拣章
- 地址和投递留白区
- 红蓝航空信边线
- 偏暖色的纸张底
- 局部留白
- 底部日期签条与条码

参考图最重要的不是“元素越多越好”，而是：

- 每个元素都像真实信件里会出现的东西
- 元素之间有层次，不会全堆在一起
- 装饰服务于“这是一张寄出的漫步记录”这一主题

## 4. 本轮美化的核心思路

本轮美化没有推翻原卡片结构，而是在现有卡片上，围绕“邮件感”做增强。

核心原则如下：

### 4.1 让卡片更像“被寄出的信件”

不是只加贴纸装饰，而是补齐信件语义里最关键的几个区域：

- 邮票区
- 投递 / 地址区
- 邮戳区
- 航空信边线
- 分拣 / 寄送标签区

### 4.2 让照片更像“夹在信里的相片”

因此对图片区的处理不是单纯铺图，而是加强：

- 白色相纸描边
- 内层细描边
- 贴胶带固定感
- 纸张叠放的收藏感

### 4.3 保持可读性，不让装饰压过内容

卡片核心仍然是：

- 漫步主题
- 当前任务
- 用户记录
- 陪伴文案
- 图片

所以装饰元素虽然增加了，但都集中在边缘、顶部、角落和底部，不直接压在正文阅读区上。

## 5. 当前已实现的美化内容

本轮已经在 `miniprogram/components/checkin-card/checkin-card.js` 中实现以下增强：

### 5.1 邮票升级

原本是简单色块邮票，现在升级为更接近真实邮票的结构：

- 外层齿孔边
- 内层彩色邮票面
- `POST`
- `66`
- 面值数字

这样左上角不再只是一个装饰块，而更像真正的寄送邮票。

### 5.2 地址区与投递留白

在卡片右上增加了邮件地址区样式：

- 半透明地址块
- 横向书写线
- `AIR MAIL NOTE` 标识

它的作用不是承载真实地址，而是给画面建立“寄送区”的视觉语义。

### 5.3 航空信红蓝边线

增加了一条横向红蓝斜线装饰，模拟航空信封边缘。

这个元素非常重要，因为它能快速让用户联想到：

- 邮件
- 航空信
- 投递中的私人记录

同时它还能自然切分顶部信息区和正文区。

### 5.4 邮戳 / 分拣章 / 标签章

增加了多组章类元素：

- 圆形邮戳
- `AIR` 标签章
- `LINE 66` 运输 / 路线标签章
- 底部 `SORT` 分拣章

这些元素的好处是：

- 让画面更像真实邮政流程中的信件
- 能制造倾斜、叠印、轻微不规则的手工质感
- 比单纯贴图更有“被使用过”的痕迹感

### 5.5 图片区强化

图片区域除了原有拼图逻辑，还增加了：

- 外层白边，模拟冲印照片
- 内层细描边，增强精致感
- 与纸张底的层次区分

这样图片区会更像“贴在卡片上的旅行照片”，而不是直接把图片插到画面里。

### 5.6 底部寄送信息强化

底部新增了：

- `BY AIR` 小签条
- 更完整的底部标签感
- 与原有条码共同构成“投递完成”区域

同时，日期显示也做了修正：

- 只保留日期，不再把时分秒塞进小方框
- 日期方框加宽，避免撑出边界

## 6. 技术实现方式

### 6.1 仍然采用前端 canvas 绘制

本轮美化没有改成服务端出图，而是继续沿用小程序前端 canvas：

- `canvas`
- `2d context`
- `wx.canvasToTempFilePath`

原因是：

- 即时性更强
- 生成成本低
- 不依赖服务端图像管线
- 便于继续按任务动态渲染

### 6.2 主要通过绘制函数拆分元素

为避免卡片逻辑越来越乱，新增元素通过独立绘制函数实现，例如：

- `drawStampWithPerforation`
- `drawAddressBlock`
- `drawAirmailString`
- `drawMailBadge`
- `drawInkStamp`

这种方式的优点是：

- 后续可以单独调某一类元素
- 不必每次都通读整张卡片的绘制代码
- 更适合继续迭代不同风格版本

### 6.3 使用轻量模拟而不是重素材堆叠

很多邮件质感并不是依赖大量 PNG，而是通过 canvas 绘制模拟出来的：

- 线条
- 圆角矩形
- 旋转章
- 半透明叠色
- 描边和阴影

这样做有几个好处：

- 更轻量
- 更容易统一配色
- 不容易因为素材太多导致性能波动

## 7. 当前效果提升点

从视觉结果上看，这轮美化主要改善了以下几个方面：

### 7.1 卡片主题更明确

现在用户第一眼会更容易理解：

- 这不是普通截图
- 也不是普通海报
- 而是一张“寄出的漫步打卡卡片”

### 7.2 收藏感更强

卡片更像一张值得保存的实体物：

- 有邮票
- 有印章
- 有寄送标签
- 有照片贴附感

这种质感比单纯 UI 装饰更适合社交分享和纪念用途。

### 7.3 视觉层次更丰富

现在卡片的层次可以分成：

- 顶部邮件元素层
- 中部记录内容层
- 图片收藏层
- 底部投递完成层

画面不再是“从上到下平铺信息”。

## 8. 仍可继续优化的地方

虽然这轮已经比之前更有质感，但还可以继续提升：

### 8.1 增加更真实的纸张细节

例如：

- 轻微折痕
- 局部压痕
- 边缘磨损
- 更自然的纸张噪点

这些元素能进一步强化“实体信件”的真实感。

### 8.2 增加更像邮政系统的字段区

例如：

- 收件人 / 寄件人块
- 邮编格
- 手写批注区
- 投递路径短码

但要注意克制，避免把卡片做成信息表单。

### 8.3 优化章的随机性

目前章的位置和角度仍偏固定。后续可以加入轻微随机：

- 不同卡片章的位置略不同
- 角度略不同
- 透明度略不同

这样每张卡片会更像“独一份”。

### 8.4 按主题切换邮寄风格

后续可以根据不同主题做细分视觉：

- 色彩主题：邮票更明亮
- 声音主题：加频谱式印章
- 动物主题：加更柔软的边角与小图标
- 气味主题：更轻的留白和暖色纸面

这样卡片不会全部长得完全一样。

## 9. 生产环境建议

如果后续要把打卡卡片作为稳定交付能力，建议继续做以下几件事：

### 9.1 固化版式规范

建议明确：

- 顶部区域高度
- 图片数量上限
- 邮票尺寸
- 印章数量上限
- 底部标签区宽度

避免后续小改动把整体比例改乱。

### 9.2 建立卡片视觉版本号

当前卡片样式会随着代码演进而变化。生产环境建议加入：

- 卡片模板版本
- 历史记录对应版本号

这样后续改版不会影响旧记录回放。

### 9.3 做低端机渲染兜底

当前卡片元素增多后，更需要关注：

- 低端机 canvas 失败
- 大图渲染耗时
- 字体兼容差异

建议为：

- 图片数量
- 分辨率
- 阴影与透明叠层数量

设置合理上限。

### 9.4 增加卡片回归检查

后续每次改卡片样式时，建议至少检查：

- 标题是否溢出
- 日期是否超框
- 地点是否与底部标签冲突
- 图片裁切是否正常
- 邮票 / 印章是否压到正文

## 10. 当前结论

当前打卡卡片已经从“基础记录卡”升级为更接近“邮寄纪念卡”的视觉方向。

本轮美化的核心价值在于：

- 明确了“邮件感”这一主视觉语言
- 用较轻量的 canvas 手段增强了卡片质感
- 没有破坏原有动态渲染逻辑
- 为后续继续做卡片版本化、主题化和生产化打下了基础

如果下一步继续优化，建议优先顺序如下：

1. 继续细化纸张与印章质感。
2. 处理不同长度标题、地点、日期的版式稳定性。
3. 做卡片模板版本号。
4. 增加低端机和长图场景的渲染兜底。
5. 再考虑更丰富的主题化视觉分支。
