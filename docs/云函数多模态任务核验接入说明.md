# 云函数多模态任务核验接入说明

## 当前实现

当前小程序已经接好这条云函数链路：

1. 用户在记录页的某个任务下，分别上传该任务自己的图片、视频、录音，并可填写任务备注
2. 点击 `AI核验并打分`
3. 小程序把该任务对应的素材上传到云存储，并调用 `verifyMission` 云函数
4. `verifyMission` 云函数会：
   - 用视觉模型分析图片和视频
   - 用音频模型分析录音
   - 再用文本模型汇总出最终评分
5. 返回结果结构：
   - `passed`
   - `score`
   - `comment`
   - `evidence`
   - `confidence`
6. 小程序把评分结果保存到 `missionReviews`
7. 点击 `完成本次漫步并保存` 后，评分结果、任务素材映射、轨迹、图片/视频/录音、文字一起写入 `walkRecords`
8. 足迹详情页会在“任务结果”里显示每个任务的 AI 分数、结论和证据标签

## 你需要手动做的事

### 1. 给 `verifyMission` 云函数配置百炼环境变量

至少需要配置：

- `AI_API_KEY`
- `AI_BASE_URL`
- `AI_VISION_MODEL`
- `AI_AUDIO_MODEL`
- `AI_TEXT_MODEL`

推荐值：

- `AI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1`
- `AI_VISION_MODEL=qwen3.5-plus`
- `AI_AUDIO_MODEL=qwen3-omni-30b-a3b-captioner`
- `AI_TEXT_MODEL=qwen-plus`

说明：

- `AI_API_KEY` 使用阿里云百炼的 API Key
- 如果你的账号没有开通上面的模型，可以把模型名换成你账户里可用的等价模型

### 2. 重新上传云函数

需要重新部署至少这两个云函数：

- `verifyMission`
- `createWalk`

如果你是通过微信开发者工具部署：

1. 右键 `cloudfunctions/verifyMission`
2. 选择“上传并部署：云端安装依赖”
3. 对 `cloudfunctions/createWalk` 重复一次

### 3. 确认云开发环境权限正常

要确保：

- 小程序当前使用的 `cloudEnvId` 是正确的
- 云函数能访问云存储里的临时文件 URL
- 录音文件格式保持为 `mp3`

### 4. 用新记录验证

旧的历史记录不会自动补出任务评分，因为之前没存这些字段  
需要重新保存一条新的漫步记录来验证

## 当前数据结构

`walkRecords` 中现在会额外保存：

- `missionReviews`
- `missionAssetMap`
- `videoList`
- `audioList`
- `trackStartedAt`
- `trackStoppedAt`
- `routeStats`

其中：

- `missionReviews` 保存每个任务的评分结果
- `missionAssetMap` 保存每个任务对应的图片、视频、录音、任务备注

## 建议测试顺序

1. 进入记录页，展开某个任务
2. 只给这个任务上传一张图片，点 `AI核验并打分`
3. 看是否返回分数和评论
4. 再给另一个任务上传录音或视频，再次核验
5. 最后保存整次漫步
6. 去足迹详情页确认：
   - 任务结果是否有分数
   - 图片/视频/录音是否都能显示
   - 时间和里程是否还在
