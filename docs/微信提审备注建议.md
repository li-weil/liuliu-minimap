# 微信提审备注建议

更新时间：2026-04-03

## 建议填写版本

以下内容可直接作为微信小程序提审备注的基础版本使用。

## 提审备注正文

本小程序为城市漫步记录工具，主要功能为：

- 用户在探索页选择当前位置、搜索地点或手动设定探索点
- 系统根据地点与模式生成本次漫步主题和任务
- 用户在记录页完成任务打卡，记录图文、视频、录音等内容
- 用户可查看自己的历史足迹、任务记录和纪念内容

关于定位能力说明：

- 小程序当前使用的是前台实时定位能力
- 用户仅在记录页主动点击“开启轨迹追踪”后，才会开始记录本次漫步轨迹
- 轨迹功能仅用于本次漫步路线记录，不涉及后台持续定位
- 页面内已明确提示用户：轨迹追踪需尽量保持小程序停留在前台，切到后台后连续轨迹可能中断

关于隐私授权说明：

- 小程序已接入用户隐私保护指引授权链路
- 在定位、上传图片/视频、录音、保存到相册、保存个人资料等敏感能力触发前，会先向用户展示用途说明
- 用户阅读并同意后，才会继续相关能力调用

关于图片、视频、录音与相册能力说明：

- 图片、视频、录音仅在用户主动点击上传或录制时使用
- 摄像头仅在用户主动拍摄任务打卡照片或视频时使用
- 相册权限仅在用户主动保存纪念卡片或贴纸到本地时使用
- 不会在无操作情况下主动调用以上能力

关于账号资料说明：

- 用户可填写昵称、选择头像，用于保存个人足迹及在同行模式中标识成员身份

关于同行模式说明：

- 同行模式为邀请制协作记录功能
- 仅受邀成员可进入同一个房间共同完成漫步任务
- 不提供开放式社区讨论广场

## 精简版

本小程序为城市漫步记录工具。用户可选择地点生成漫步主题，在记录页完成任务打卡并保存图文、视频、录音等内容。轨迹记录当前仅使用前台实时定位，用户需主动点击“开启轨迹追踪”后才会记录本次漫步路线，不涉及后台持续定位。图片、视频、录音和相册能力均仅在用户主动操作时使用。同行模式为邀请制协作记录，不提供开放式社区讨论功能。

如果需要强调隐私整改，可补一句：

小程序已接入用户隐私保护指引授权链路，在定位、拍摄、上传、录音、保存到相册等能力触发前，会先向用户说明用途并征得同意。

## 提审演示路径建议

建议在审核后台补充演示路径，方便审核员快速走通：

1. 进入“探索”页
2. 点击“当前定位”或手动地图选点
3. 生成一个主题并点击“开始这次漫步”
4. 在“记录”页点击“开启轨迹追踪”
5. 查看按钮下方文案，可见“当前使用前台实时定位”的说明
6. 在定位、上传图片/视频、录音或保存到相册时，可见隐私用途说明弹窗
7. 继续完成任务打卡并保存
8. 到“足迹”页查看历史记录

## 备注使用建议

- 如果你提审时已经关闭公开分享或弱化多人模式，可以删掉备注里的相关描述，让口径更收敛
- 你已经补了隐私授权代码链路，若后台隐私保护指引也已保存发布，可以追加一句“已按平台要求配置用户隐私保护指引与授权链路”
- 如果审核员重点追问定位能力，优先强调“前台实时定位”“用户主动开启”“仅用于本次漫步轨迹记录”“不涉及后台持续定位”
- 如果审核员重点追问隐私合规，优先强调“敏感能力触发前有用途说明弹窗”“用户同意后才继续调用”“后台隐私保护指引已按实际收集项配置”
