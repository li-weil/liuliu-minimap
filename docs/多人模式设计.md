# 多人模式设计

## 1. 当前定位

多人模式现在就是第一期的 `同行模式`：几个人围绕同一个探索点和同一张主题任务票据出发，各自提交素材，结束后生成一份团队结果。

它不是聊天群、实时定位社交、PK 排行榜，也不是陌生人匹配。它只服务 `liuliu` City Walk 的主流程：开始漫步、完成任务、回看纪念。

## 2. 当前已实现方案

### 2.1 组队与邀请

入口在探索页：

- 用户先设置探索点。
- 生成主题任务。
- 出发方式选择 `同行`。
- 点击 `发起同行漫步` 创建房间。
- 创建成功后进入 `team-room` 房间页。
- 通过微信小程序原生分享邀请好友进入 `team-join`。

未登录用户会先去个人页完善资料。登录成功后，再回到原来的加入房间路径。

### 2.2 房间与权限

房间状态：

- `waiting`：待出发，可邀请、加入、退出或由房主解散。
- `active`：进行中，不允许新成员加入，成员进入团队记录页提交内容。
- `finished`：已结束，只允许回看团队结果。
- `dissolved`：历史兼容状态；新解散房间会归档为 `finished`。

权限边界：

- 只有房主可以开始同行。
- 只有房主可以结束同行。
- 未加入用户只能看基础预览，不能读取完整成员、贡献和动态。
- 已结束同行记录可由当前成员从自己的纪念卡册删除，不影响其他成员。

### 2.3 团队任务

任务采用共享模型：

- 全队看到同一份任务列表。
- 任意成员可以对任意任务提交文字、图片、视频、录音。
- `同步给团队` 即为该任务的打卡动作。
- 团队完成度按任务聚合，只要某任务有任意成员完成，就记为团队完成。

还没有分工任务、协作任务、任务角色或完成阈值配置。

### 2.4 团队记录与草稿

团队记录页已支持：

- 任务切换。
- 我的提交区。
- 图片、视频、录音、文字素材。
- 上传进度。
- 团队动态流。
- 房主结束同行。

记录页有本地草稿保护：

- 按 `roomId + userId + mission` 保存未同步草稿。
- 页面轮询不会覆盖当前用户带内容的 dirty 草稿。
- 退出后重新进入同一房间，可恢复当前设备当前用户未同步草稿。
- 房主结束前会检查自己和成员是否仍有带内容但未同步的任务。

### 2.5 轻动态

动态流只展示系统事件，不是自由聊天。

已覆盖事件包括：

- 房间创建。
- 成员加入。
- 成员退出。
- 房主开始同行。
- 成员更新或完成任务。
- 房主结束同行。

这样能让用户感到队友也在参与，同时不会把产品变成聊天工具。

### 2.6 团队结果

团队详情页展示：

- 团队主题。
- 探索点。
- 成员。
- 团队统计。
- 任务回顾。
- 每个任务下的成员贡献。
- 图片、视频、录音。
- 团队动态。
- 任务级团队打卡卡片。

团队打卡卡片会汇总所有成员的贡献。首次手动生成后，图片会上传并写入 `missionCardMap`；之后打开时优先复用已保存图片。

## 4. 当前不做的内容

当前版本不做：

- 自由聊天。
- 实时语音。
- 精确实时位置共享。
- 陌生人匹配。
- PK 排行榜。
- 常用搭子关系链。
- 团队长期群组。
- 分工任务和强协作任务。

这些能力可以以后再评估，但不应影响同行主流程的稳定性。

## 5. 推荐下一步

### 5.1 优先加固上线基础

下一步先补工程和运维稳定性：

- 补齐团队集合初始化和云函数部署检查清单。
- 固化 `teamWalkRooms / teamWalkMembers / teamWalkContributions / teamWalkActivities` 字段 schema。
- 明确每个房间状态允许的操作，减少前后端状态判断漂移。
- 真机验证团队草稿在弱网、页面回收、多账号切换下的恢复表现。
- 为团队上传、提交、结束、卡片生成补更明确的失败重试提示。

### 5.2 缩短恢复路径

进行中的同行可以从足迹详情页恢复，但路径偏长。

推荐下一步：

- 在启动提醒里提供直达最新进行中同行的按钮。
- 在足迹列表卡片上直接露出 `继续同行` 动作。
- 对 `waiting` 房间露出 `继续组队`，对 `active` 房间露出 `重新进入`。

### 5.3 增强团队纪念感

团队详情页已经能回看内容，但纪念感还不够强。

推荐下一步：

- 增加团队高光总结区。
- 按任务或成员精选代表素材。
- 给团队结果页生成一张总览纪念卡。
- 保留任务级卡片，不再每次打开重复生成。

### 5.4 增强轻互动，但不做重聊天

推荐优先增强动态流：

- 让动态按任务聚合。
- 标出“谁点亮了哪个任务”。
- 对任务贡献增加轻量反应或短反馈。
- 继续避免自由聊天和实时位置直播。

### 5.5 后续玩法评估

共享任务跑稳后，再评估：

- 双人默契任务。
- 抽签分工任务。
- 限时协作任务。
- 团队角色。
- 团队成就。
- 主题活动赛季。

这些玩法都应建立在三个前提上：同行流程稳定、团队纪念产物有吸引力、用户愿意发起和加入。
