# AI打卡贴纸接入说明

## 当前实现

当前小程序已经补齐这条流程：

1. 记录页新增 `生成打卡贴纸`
2. 点击后，小程序内部会自动走两段式：
   - 先生成小诗与贴纸方案
   - 再生成贴纸图片
3. 第一步会调用云函数 `generateSticker`
4. 云函数先用豆包文本模型生成：
   - 一句短诗
   - 小标题
   - 贴纸视觉 prompt
5. 第二步再单独调用同一个云函数的生图阶段
6. 再用豆包图像生成模型生成贴纸底图
7. 云函数把底图上传到云存储，返回 `cloud://` 文件地址
8. 小程序在记录页预览贴纸，并把小诗以叠加艺术字方式展示在贴纸上
9. 点击保存后，贴纸会跟着漫步记录一起写入 `walkRecords`
10. 足迹详情页会展示这张贴纸

## 当前默认模型

当前配置文件默认使用：

- 文本模型：`doubao-seed-1.6-flash`
- 图像模型：`doubao-seedream-4-0-250828`

说明：

- 文本模型用来生成小诗和贴纸 prompt
- 图像模型用来生成贴纸底图
- 小诗不是直接交给模型写到图片里，而是在前端叠加显示，这样中文更稳定

## 你需要手动做的事

### 1. 修改配置文件

只需要改：

[config.js](D:/liuliu-minimap/cloudfunctions/generateSticker/config.js)

至少把：

- `apiKey`

改成你自己的火山方舟 API Key

如果你的控制台里模型名不同，也可以一起改：

- `textModel`
- `imageModel`

### 2. 部署云函数

右键部署：

- [generateSticker](D:/liuliu-minimap/cloudfunctions/generateSticker)

选择：

- `上传并部署：云端安装依赖`

### 3. 重新编译小程序

因为前端新增了贴纸入口和保存字段，所以要重新编译一次

## 建议测试顺序

1. 进入探索页生成一个主题
2. 进入记录页
3. 点击 `生成打卡贴纸`
4. 点 `生成打卡贴纸`
5. 等待小诗和贴纸图片都完成
6. 看是否出现贴纸预览
7. 点击 `完成本次漫步并保存`
8. 去足迹详情页确认是否能看到贴纸

## 注意事项

- 贴纸底图由云函数生成并上传到云存储，旧记录不会自动补出贴纸
- 如果方舟控制台里没有默认模型，请把 `config.js` 中的模型名替换成你账号当前可用的豆包模型
- 如果生成失败，优先检查 `apiKey`、模型权限和云函数部署是否成功
